## 1. 项目核心定位

### 1.1 一句话定义

**PawStar（爪星）** 是一款 iOS 应用，帮用户上传宠物照片，1 分钟生成各类"萌系证件"（身份证 / 驾驶证 / 毕业证等），让铲屎官以仪式感+幽默感的方式展示自己的宠物。

### 1.2 产品故事

> "你家的毛孩子是家里的小明星 ⭐ 现在它有自己的证件了。"

### 1.3 项目代号

- 英文名：**PawStar**
- 中文名：**爪星 · 萌宠证件馆**
- 副标题（30 字内）：**给家里的小明星办张证 ⭐**

## 3. 目标用户

### 3.1 核心用户画像

- **年龄**：18-35 岁
- **性别**：女性为主（70%+），男性次之
- **身份**：有宠物的年轻人，特别是单身女性、新手父母、Z 世代铲屎官
- **平台行为**：重度小红书用户、爱发朋友圈、热衷"晒娃"（宠物）
- **付费意愿**：对宠物相关消费付费意愿强；对萌系/仪式感产品付费门槛低

### 3.2 用户痛点 / 动机

1. **晒宠物没有新玩法**：朋友圈晒了几年宠物，缺新鲜的呈现形式
2. **仪式感稀缺**：希望表达"我家毛孩子是家里的小宝贝"
3. **社交货币需求**：能在小红书 / 朋友圈引发讨论的内容
4. **轻度娱乐**：5 分钟解压、不需要思考、即时满足

### 3.3 不是目标用户

- ❌ 真正需要宠物医疗记录的用户（已有专业产品）
- ❌ 想给宠物办真证件出境的用户（这是法规问题，不是我们要做的）

## 4. 核心功能范围

### 4.1 MVP 范围（V1，2-3 周开发）

**核心闭环**：

```
拍照/选照片 → 填信息 → 生成证件 → 保存/分享
```

**必做功能**：

| 功能 | 优先级 | 描述 |
|------|-------|------|
| 首页：办证大厅 | P0 | 卡片式展示 3 种证件入口 |
| 选择宠物照片 | P0 | 调用相册或相机 |
| 照片裁剪 | P0 | 方形裁剪，模拟证件照 |
| 填写宠物信息 | P0 | 姓名、品种、生日等基础字段 |
| 生成身份证 | P0 | 第一个证件类型 |
| 生成驾驶证 | P0 | 第二个证件类型 |
| 生成毕业证 | P0 | 第三个证件类型 |
| 预览 + 保存到相册 | P0 | 高清 PNG 输出 |
| 原生分享面板 | P0 | 一键分享到微信/小红书 |
| 历史证件本 | P1 | 本地存储已生成的证件 |
| 梗文案随机生成 | P1 | 每次办证自动填入有梗的字段 |

**明确不做**：

- ❌ 用户系统 / 登录注册
- ❌ 云端同步
- ❌ 内购订阅（V1.5 才加）
- ❌ AI 抠图（V2 才加）
- ❌ 多语言（先做中文，V2 国际化）
- ❌ Apple Watch / Widget（V2 考虑）

### 4.2 V1.5 范围（上线 1 个月后）

| 功能 | 描述 |
|------|------|
| Pro 版购买 | ¥1 一次性内购，去水印 |
| 新增 3 种证件 | 护照、结婚证、毕业证 Pro 版 |
| 自定义颜色 | 证件背景色、边框色可选 |
| 高清打印导出 | 适配实体卡尺寸 |

### 4.3 V2 范围（3-6 个月后）

- AI 自动抠图（VisionKit 或 GPT-4V）
- 多宠物管理（最多 5 只）
- 订阅制（高级功能解锁）
- 实体卡邮寄（小红书电商对接）
- 出海英文版

### 5.3 关键技术决策

- **不引入任何第三方依赖**（V1 阶段）：纯原生 API，减少审核风险
- **完全本地化**：不联网、不传输用户数据（隐私优势 + 审核优势）
- **不接入分析 SDK**（V1）：先求上线，统计后期再补

## 6. 设计系统

### 6.1 设计哲学

**Playful Luxury（俏皮的轻奢）**

- 既要有"证件"的庄严感（让用户觉得办得正经）
- 又要有"萌宠"的可爱感（让用户想分享）
- 避免：紫色渐变 / 居中大 emoji / 通用 Material 风格（这些是 "AI slop"）

### 6.2 颜色系统

```
primary       = #FF8A3D   // 主橙（爪印感）
primaryDark   = #E5701F   // 深橙
star          = #4A6FFF   // 星空蓝（明星感）
gold          = #FFCB47   // 证件金色装饰
bgWarm        = #FFF8F0   // 暖白背景
bgCard        = #FFFFFF   // 卡片白
textPrimary   = #1A1A1A   // 主文字
textSecondary = #6B6B6B   // 次文字
border        = #EDEDED   // 边框灰
```

### 6.3 字体系统

- **标题（数字 / 英文）**：SF Pro Display Bold
- **标题（中文）**：PingFang SC Semibold
- **正文**：SF Pro Text + PingFang SC Regular
- **证件号 / 等宽字段**：SF Mono 或 JetBrains Mono

### 6.4 证件设计规范

每个证件模板必须包含的元素：

1. **明显的"娱乐用途"标识**：右上角小字"Toy Certificate · 仅供娱乐"（**苹果审核合规要点**）
2. **不与官方证件 100% 一致**：调整证件号位数、颜色、版式
3. **手绘感装饰**：小爪印、星星、心形等萌系元素
4. **可分享时的水印**（V1 免费版）：底部小字"made with PawStar"

#### 6.4.1 身份证模板

- 仿照中国二代身份证布局，但调整尺寸和颜色（避免 100% 一致）
- 字段：姓名、性别、民族（橘猫族 / 田园犬族等）、出生、住址（本铲屎官的心脏 / 主人怀里）、证件号（自动生成 18 位伪号）

#### 6.4.2 驾驶证模板

- 仿照 C 类驾照
- 字段：姓名、性别、国籍（喵星 / 汪星）、住址、出生日期、初次领证日期、准驾车型（纸箱 / 主人腿 / 航空箱）、有效期（永久）

#### 6.4.3 毕业证模板

- 仿照大学毕业证
- 字段：姓名、出生日期、毕业院校（本铲屎官猫狗大学）、专业（拆家学 / 撒娇学 / 装睡专业 / 偷食学）、成绩（优秀 / 良好 / 通过）、毕业作品（撕烂三个抱枕 / 偷吃主人 8 次外卖）

## 8. 风险与合规

### 8.1 必须遵守的规则（苹果审核）

1. **每张证件都必须有"娱乐用途"明显标识**（NOT REAL DOCUMENT 或"仅供娱乐"）
2. **不能 100% 模仿官方证件**：调整颜色、版式、号码格式
3. **不能在 App 描述中暗示"可作为身份证明"**
4. **隐私政策必须明确**：照片仅本地处理，不上传服务器
5. **包含"恢复购买"按钮**（V1.5 加内购后）
6. **苹果新规**：如果使用第三方 AI（V2 才考虑），必须明确披露数据传输

### 8.2 已知风险与应对

| 风险 | 应对 |
|------|------|
| 苹果审核因"模仿官方证件"被拒 | 添加明显娱乐标识 + 调整版式差异度 |
| 被抄袭 | V1 上线 1 个月内必须完成 V1.5（Pro 版），用差异化拉开 |
| 用户上传违规图片 | V1 不联网，避免 UGC 风险；V2 加 AI 时考虑本地审核 |
| 商标 | 上架前完成「PawStar」「爪星」中美商标查询 |

## 附录 A：用户故事示例（写代码时参考）

### Story 1：第一次使用
> 小红是新用户，下载 App 后打开首页，看到三个证件大卡片。她点击「身份证」，App 引导她选一张猫的照片，填入猫的名字「糖糖」、品种「橘猫」、生日。App 自动填入了搞笑的"民族：橘猫族"、"住址：本铲屎官的心脏"。她点击生成，2 秒后看到一张漂亮的身份证。她按"保存到相册"，然后按"分享"，把图片发到了小红书。

### Story 2：回访用户
> 小红已经办过身份证，再次打开 App。她想给猫办驾照。从首页点驾驶证 → 填表 → 生成 → 分享。她看到「历史证件本」里已有 2 张证件，可以重新查看或再分享。

### Story 3：分享传播
> 小红的朋友小芳在朋友圈看到她发的猫身份证，被"民族：橘猫族"逗笑，问她"这是用什么做的"。小红回答"PawStar"。小芳下载并体验。这就是核心传播链。
